# Manuale Utente — CPM → P6 v16

**Cosedil PMO · Bridge Budget vs SIL vs P6**

---

## Panoramica

Lo strumento **CPM → P6 v16** esegue il mapping automatico tra i dati di avanzamento economico CPM (SIL) e le attività di Oracle Primavera P6. Tutto il calcolo avviene nel browser: nessun dato viene inviato a server esterni.

**Flusso logico in tre passi:**

```
SIL Articolo → Budget Articolo → Cod. WBS
Cod. WBS     → Export P6 wbs_id → Lista Attività P6
SIL Allocato → distribuito tra attività (proporzionale a act_cost o % fisica)
```

---

## File richiesti

| File | Etichetta | Obbligatorio | Foglio atteso |
|------|-----------|:---:|---------------|
| Budget CPM | 💰 Budget CPM | **Sì** | BUDGET |
| SIL Diretti | 📊 SIL Diretti | **Sì** | SIL diretti |
| Export P6 | 📅 Export P6 | **Sì** | EXPORT_P6 |
| SIL Indiretti | 📋 SIL Indiretti | No | SIL indiretti |

Formati accettati: `.xlsx`, `.xls`, `.csv`, `.txt`
Limite dimensione: **30 MB** per file
Limite righe: **50.000 righe** per file (troncamento automatico oltre)

---

## Struttura colonne attese

**Budget CPM** — `Cod. WBS`, `Articolo`, `Importo Costo (€)`

**SIL Diretti / Indiretti** — `Cod. S.I.L.`, `Articolo`, `Importo`

**Export P6** — `task_code` (o `Activity ID`), `wbs_id` (o `WBS Code`), `act_cost`, `calc_phys_complete_pct`, `status_code`, `act_name`

---

## Procedura di utilizzo

### 1. Caricamento file

Trascina i file nelle aree (drag & drop) oppure fai clic per aprire il selettore. Dopo il caricamento la pillola corrispondente diventa verde e compare l'anteprima delle prime righe. Il pulsante **▶ Calcola Bridge** si attiva quando sono caricati i tre file obbligatori.

### 2. Calcolo

Premi **▶ Calcola Bridge**. La barra di avanzamento mostra le fasi:

| % | Fase |
|---|------|
| 10% | Parsing Budget CPM |
| 30% | Parsing SIL |
| 50% | Parsing Export P6 |
| 60% | Aggregazione SIL per articolo |
| 75% | Mapping Articolo → WBS |
| 85% | Distribuzione WBS → Activity P6 |
| 95% | Calcolo KPI e Alert |
| 100% | Completato |

### 3. Lettura risultati

I risultati si articolano in quattro schede:

#### Scheda Riepilogo WBS

Tabella aggregata per codice WBS con colonne: WBS, Des. WBS, SIL (€), P6 Costo (€), Delta (€), N. Att., Budget (€), Status. Riga **TOTALE** in fondo.

#### Scheda Distribuzione P6

Dettaglio per singola attività P6 con metodo di distribuzione:

| Tag | Metodo | Quando applicato |
|-----|--------|-----------------|
| COST | Proporzionale a act_cost | act_cost totale WBS > 0 |
| PHY | Proporzionale a % fisica | act_cost = 0 ma phys% > 0 |
| EQ | Quota uguale tra attività | Entrambi i valori = 0 |

#### Scheda Alert

- **⚠️ WARN** — attività con SIL assegnato ma % fisica = 0 (e non "Not Started")
- **🚨 ERR** — articoli SIL non trovati nel Budget CPM
- **🚨 ERR** — WBS presenti nel SIL ma assenti dall'Export P6

#### Scheda Deviazioni

Attività il cui scarto |SIL − P6 Costo| supera la soglia configurata. Colonne: WBS, Activity ID, Delta, Valore assoluto, % sul P6 Costo.

---

## KPI principali

| KPI | Descrizione |
|-----|-------------|
| SIL Allocato (€) | Totale SIL correttamente mappato su attività P6 |
| P6 Costo (€) | Somma act_cost di tutte le attività mappate |
| Budget CPM (€) | Totale budget da file CPM |
| CPI | Cost Performance Index = SIL Allocato / P6 Costo |
| SIL Mappato % | Quota del SIL totale che ha trovato corrispondenza in P6 |

Il **CPI** è semaforo: verde se 0,95 ≤ CPI ≤ 1,05 · arancio se CPI > 1,05 · rosso se CPI < 0,95.

---

## Messaggi di stato

| Banner | Significato |
|--------|-------------|
| ⚠ Budget CPM mancante | Nessun Budget caricato — il mapping non è possibile |
| 🚨 X art. non in Budget + Y WBS non in P6 | Articoli o WBS senza corrispondenza |
| ✓ Bridge completo | Tutti gli articoli SIL mappati correttamente |

---

## Note operative

- L'Export P6 accetta sia il formato standard Primavera (`task_code`, `wbs_id`) sia il formato report esteso (`Activity ID`, `WBS Code`): conversione automatica.
- Il matching Articolo → WBS è case-insensitive con normalizzazione spazi.
- I dati non vengono mai inviati fuori dal browser (CSP `connect-src 'none'`).
- Reset: cancella tutti i file e i risultati; la soglia di deviazione viene mantenuta.
  
# Configurazione — CPM → P6 v16

**Cosedil PMO · Parametri tecnici e personalizzazione**

---

## Soglia Deviazione

**Campo:** `Soglia Deviazione (€)` — pannello sinistro, sezione *Configurazione*

Controlla quale scarto minimo tra SIL allocato e P6 Costo viene segnalato nella scheda **Deviazioni**.

| Parametro | Valore |
|-----------|--------|
| Predefinito | 5.000 € |
| Minimo | 100 € |
| Passo | 500 € |
| Massimo | 10.000.000 € |

La soglia viene salvata in `localStorage` (chiave `bridge_threshold`) e ripristinata alla riapertura. Un'attività compare nelle Deviazioni se `|SIL − P6 Costo| ≥ soglia`.

## Colonne riconosciute

Le intestazioni vengono cercate nelle prime 12 righe; match su sottostringa case-insensitive.

**Budget CPM** — `cod. wbs` · `articolo` · `importo costo` · `costo` · `budget` · `wbs`

**SIL Diretti / Indiretti** — `cod. s.i.l.` · `articolo` · `importo` · `sil` · `costo`

**Export P6** — `task_code` · `wbs_id` · `act_cost` · `calc_phys_complete_pct` · `status_code` · `act_name`

Per Export P6 è supportata anche la variante con intestazioni estese (`Activity ID`, `WBS Code`, `Actual Total Cost`, `Physical % Complete`, ecc.): conversione automatica prima del parsing.

---

## Normalizzazione WBS

I codici WBS vengono normalizzati prima di qualsiasi confronto:

- Conversione in uppercase, trim
- Split sul carattere `.`
- Segmenti **numerici puri**: rimossi gli zeri iniziali (`01` → `1`)
- Segmenti **alfanumerici**: mantenuti invariati

| Input | Normalizzato |
|-------|-------------|
| 01.002.A | 1.2.A |
| WBS.010 | WBS.10 |
| A.B.C | A.B.C |

---

## Normalizzazione Articolo

- Conversione in lowercase
- Collasso spazi multipli
- Trim iniziale e finale

Il matching Articolo → WBS usa tre livelli di fallback:

| Priorità | Metodo | Dettaglio |
|----------|--------|-----------|
| 1 | Match esatto | Articolo normalizzato |
| 2 | Match prefisso | Primi 8 caratteri |
| 3 | Match fuzzy | Sottostringa 5 caratteri |

---

## Metodi di distribuzione SIL → Attività P6

Quando una WBS ha più attività P6 associate:

| Priorità | Metodo | Condizione | Tag |
|----------|--------|------------|-----|
| 1 | Proporzionale a act_cost | Somma act_cost WBS > 0 | COST |
| 2 | Proporzionale a % fisica | act_cost = 0 e somma phys% > 0 | PHY |
| 3 | Quota equa | Tutti i valori = 0 | EQ |
